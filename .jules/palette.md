## 2024-03-08 - Added title attribute to programmatically focusable elements
**Learning:** In the completions panel webview (`suggestionsPanelWebview.ts`), `<pre>` blocks inside `.snippetContainer` are dynamically assigned `tabIndex = 0` to support keyboard navigation (via the `navigateNextSolution` and `navigatePreviousSolution` functions which call `.focus()`). However, assigning `tabIndex=0` to non-interactive structural elements can create confusing stops for screen reader users if there's no descriptive context (or if it overrides the content itself with a generic aria-label). Using `title='Use arrow keys to scroll'` provides a helpful tooltip for sighted users and some context for screen readers without replacing the actual code snippet content.
**Action:** When dynamically making structural elements (like `<pre>` or `<div>`) keyboard focusable with `tabIndex = 0` for scrolling/navigation purposes, add a `title` attribute to provide context. Avoid `aria-label` here if it risks overriding the textual content (like code snippets) that the screen reader should actually read.
